- [x] Failed-unlock rate limiting: implemented in `firmware/src/gesture/gesture.c`. `CONFIG_CANTIL_UNLOCK_RATE_LIMIT_THRESHOLD` (default 3) free attempts trigger only the `FAIL` blink; the next failure begins a growing lockout (10 s → 30 s → 2 min → 10 min → 1 h, capped). Counter persists to `/lfs/config_unlock_attempts.bin` via `storage_unlock_attempts_*`, clears to 0 on successful unlock, and is wiped on `RESET_DEVICE`. The active lockout-until timer is uptime-relative and is intentionally dropped across reboot — but the next wrong attempt re-applies the same-tier lockout, so power-cycling can only restart the current tier, not skip it. Taps that arrive during an active lockout are dropped at the entry point with `LED_PATTERN_BUSY_REJECT` instead of `TAP_ACK`; the LOCKED idle loop keeps running so the user can still see the device's state.
