# nrfutil 5.2.0 pins pyyaml~=4.2b1, but PyYAML 4.x calls
# collections.Hashable which was removed in Python 3.10+ — the venv runs on
# 3.14, so Zephyr's build (list_hardware.py) crashes during board lookup with
# that pin.  Force a modern PyYAML; nrfutil's imports still work fine.  Pin
# applied after nrfutil so pip's resolver records the override.
pyyaml>=6.0.1
