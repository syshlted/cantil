# Cantil

- Certificate Authority, Noise Transport, Isolated and Local
- Compact Authority on Nano Trusted Integrated Logic
- CANTIL: A Nifty Trustworthy Identity Locker
- CANTIL Authorizes Networking Through Insecure Links
- CANTIL Ain't No Trivial, Insecure Lemon

**Tiny PKI. Big potential.**

*Purpose-built hardware CA for embedded device identity. Keys are generated in hardware, encrypted at rest, and never leave the device — provisioning happens over an encrypted local channel, with no network interface and no key export.*

---

- **Real PKI, fully offline.** Root and intermediate CAs live on the device. Private keys are generated by a hardware TRNG, stored encrypted on the device's flash, and used inside the crypto accelerator — they never appear in host memory.
- **An embeddable CA for your other projects.** `libcantil` lets a constrained device call out to a Cantil instance for certificate operations. Your firmware stays small; your PKI stays consistent.
- **Untrusted transports welcome.** Every byte between client and device is Noise_XX-encrypted with mutual static-key authentication. USB sniffing, BLE eavesdropping, and MITM all see ciphertext only.

**Current platform support:** any nRF52840 board. Reference target: [Seeed Studio XIAO BLE Sense](https://wiki.seeedstudio.com/XIAO-BLE/). The CA, gesture state machine, Noise session, and CBOR protocol are platform-agnostic — new SoCs slot in behind the existing Kconfig and devicetree seams.
